## WorldChatMiniPanel 聊天窗组件改造总结

### 解决的问题
1. **删除全局提示层 GameGlobalHintLayer** - 背包、伙伴等页面不再显示左下角小窗，只保留房车和战场板块底部的 WorldChatMiniPanel
2. **物品名颜色编码** - 提示文本中物品名根据稀有度着色（common灰色/uncommon绿色/rare蓝色/epic紫色/legendary金色/unique品红），与背包格子边框颜色一致
3. **文本左对齐** - 使用 Align(alignment: Alignment.centerLeft) 包裹 Text.rich 解决单行文本居中问题
4. **背景透明度统一** - 提取 WorldChatMiniPanelLayout.backgroundAlpha 常量，两端渐变使用同一值
5. **底部导航栏遮挡** - 添加 gapAboveNavBar 间距，聊天窗浮在导航栏上方
6. **收折闪烁问题** - 用 Align(alignment: Alignment.bottomCenter) 包裹 ListView，内容从底部往上排列，避免高度变化时先顶部布局再滚动到底部的闪烁
7. **收折逻辑简化** - 删除 _CollapsedLatestPreview，展开/收折使用相同 ListView，只改变高度

### 关键文件修改
- `lib/widgets/feedback/world_chat_mini_panel.dart` - 主组件
- `lib/controllers/game_hint_controller.dart` - 新增 itemStackAcquireRarity 字段
- `lib/controllers/player_inventory_controller.dart` - 传入物品稀有度
- `lib/main_game.dart` - 删除 GameGlobalHintLayer 引用
- 删除 `lib/widgets/feedback/game_global_hint_layer.dart`

### 技术要点
- Text.rich + TextSpan 实现多色文本
- Align + ListView(reverse: true) 实现底部锚定聊天布局
- AnimatedContainer + clipBehavior: Clip.hardEdge 平滑高度动画
- 常量提取到 Layout 类避免魔法值

## DeadLine 战斗系统审查经验

### 关键发现：onFatalDamage 触发链中的 attacker/defender 语义反转

在 `_runTriggerSkillsForEntity(owner, triggerType, contextEntity)` 中：
- `owner` = 技能拥有者（pipeline 中的 attacker）
- `contextEntity` = 技能目标（pipeline 中的 defender）

当 `onFatalDamage` 触发时：
- `owner = defender`（濒死者）
- `contextEntity = attacker`（攻击者）
- 所以在 SkillExecutor 中：`attackerState = 濒死者`，`defenderState = 攻击者`
- `defenderCurrentHp` = 攻击者的 HP（不是濒死者的！）
- `attackerCurrentHp` = 濒死者的 HP

**规则**：在 onFatalDamage/onDamaged 等防御性触发中，自身 HP 应检查 `attackerCurrentHp`，而非 `defenderCurrentHp`。

### 多目标事件架构
- MultiTargetDamageEvent/AuraApplyEvent 在 EventApplicator 层静默返回原状态
- 实际处理在 BattleEventApplier 层通过回调
- _handleMultiTargetDamage 需追踪 currentAtkState 并循环结束后回写
- _handleAuraApply 同样需要追踪攻击者状态

### Map 哈希实现
- XOR 方式对顺序不敏感，与 Dart Map 的确定性遍历顺序兼容
- 但使用 SplayTreeMap 等不同实现时可能产生 hashCode 不一致

## 伙伴名册组件重构 (2026-04-19)

### 解决的问题
1. 将伙伴页面名册抽取为通用组件 `CompanionRosterSheet`
2. 添加选中角色 overlay 效果（左上角勾选图标）
3. 修复卡片右上角 badge 溢出遮挡相邻卡片
4. 修复 `_CompanionStarBadge` 内部 Row 溢出 0.114 像素
5. 修复 `CompanionStripPortraitCard` Column 底部溢出 0.856 像素

### 技术方案
- **通用组件**: `companion_roster_sheet.dart` 提供 `onPickMemberIndex` 回调和 `selectedMemberIndex` 属性
- **选中效果**: 左上角 `Positioned` + 圆形 rust 色背景 + check icon
- **网格布局**: `GridView.builder` + `SliverGridDelegateWithFixedCrossAxisCount`，列数自适应 (3-6列)
- **间距配置**: `crossPad = 12.r` (左右内边距), `crossGap = 1.w` (列间距), `mainAxisSpacing = 0.5.h`
- **溢出修复**: 
  - 星级徽章用 `FittedBox(scaleDown)` 包裹 Row
  - 卡片整体用 `FittedBox(scaleDown)` 包裹 Column

### 避坑点
- `clipBehavior: Clip.none` 允许 badge 溢出卡片边界，需确保 `crossGap` 足够
- `ConstrainedBox` 的 `maxWidth` 限制与内部 Row 自然宽度冲突时用 `FittedBox`
- `LayoutBuilder` 中 `constraints.maxHeight` 可能为 `double.infinity`，需判断 `isFinite`

- `deadline_flutter/lib/widgets/party/companion_roster_sheet.dart`
- `deadline_flutter/lib/widgets/party/companion_strip_portrait_card.dart`
- `deadline_flutter/lib/pages/game/companions_page.dart`
